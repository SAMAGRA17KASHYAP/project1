Rebasing

In addition to merge we have a choice of rebasing

As an alternative to merge here we can do rebasing

To do above rebasing execute following commands

  1. Git checkout rightBranch
  2. Git rebase leftbranch
  3. In case of any conflicts, resolve conflicts. Add changes to staging area. And the  do                       Git rebase –continue

How rebasing internally works:

:We cannot go and simply delete branches, git internally create new commits for rightbranch with only one difference from existing one i.e of parent. What git do with existing commits, it garbage collects them if no branch is pointing to it.

Git merges preserves project history but in some cases it is not necessarily needed. Rebasing helps to refactor project history. Project using lot of rebasing, it is very likely that there history will look streamline. Rebasing history is lie but may be required.

Guideline: When  in doubt always use merge
